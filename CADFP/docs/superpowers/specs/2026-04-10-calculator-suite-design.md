# Calculator Suite Design

**Date:** 2026-04-10

**Goal**

在首页点击“计算器”后进入一张与 Figma 节点 `75:195` 对齐的计算器总页，并让 8 个 item 分别进入对应的几何计算页面，完成可输入、可计算、可展示结果的可用版本。

**Constraints**

- 保持现有 SwiftUI 应用结构，不引入新的第三方 UI 依赖。
- 入口和详情页视觉尽量与 Figma 节点 `75:195`、`86:56`、`86:123`、`86:159`、`87:255`、`87:303`、`87:339`、`88:375`、`88:407` 对齐。
- 图标资源优先复用 `Assets.xcassets/CAD看图_icon-3`。
- 交互应适配现有 `NavigationStack`，不破坏首页和已有 CAD / 水印能力入口。
- 第一版聚焦几何计算功能，不扩展历史记录、单位换算、分享导出等额外能力。

**Recommended Approach**

采用“配置驱动的计算器模块”：

- 首页只负责把 `HomeAction.calculator` 路由到新的计算器总页。
- 计算器总页负责展示 8 个卡片入口。
- 详情页使用一套通用 SwiftUI 骨架，根据几何类型配置不同的：
  - 图标资源
  - 输入字段
  - 结果字段
  - 计算公式
- 计算公式沉淀成纯 Swift 逻辑，方便用单元测试做 TDD 验证。

**Architecture**

1. 入口层

- `ContentView` 统一维护首页跳转状态。
- 点击首页“计算器”后进入 `CalculatorCatalogScreen`。

2. 展示层

- `CalculatorCatalogScreen` 还原 2 列 8 宫格。
- `CalculatorDetailScreen` 统一承接：
  - 返回按钮
  - 标题
  - 顶部图标卡片
  - 参数输入区
  - 开始计算按钮
  - 结果展示卡片

3. 领域层

- 用 `CalculatorKind` 描述 8 个工具。
- 用 `CalculatorInputField`、`CalculatorResultField` 描述字段。
- 用 `CalculatorEngine` 集中实现公式。
- 用 `CalculatorFormatter` 负责结果格式化，尽量去掉无意义的 `.0`。

4. 验证层

- 为所有公式补 `XCTest` 单元测试。
- UI 部分以编译验证和预览检查为主。

**Functional Mapping**

- `圆柱体`
  - 输入：半径、高度
  - 输出：总面积、体积
  - 公式：
    - 总面积 = `2πr(r + h)`
    - 体积 = `πr²h`

- `球体`
  - 输入：直径
  - 输出：球体表面积、球体体积
  - 公式：
    - 半径 = `d / 2`
    - 表面积 = `4πr²`
    - 体积 = `4/3 × πr³`

- `立方体`
  - 输入：边长
  - 输出：面积、体积
  - 公式：
    - 面积 = `6a²`
    - 体积 = `a³`

- `圆锥`
  - 输入：半径、高度
  - 输出：斜面高、曲面表面积、圆锥表面积、圆锥体积
  - 公式：
    - 斜面高 = `sqrt(r² + h²)`
    - 曲面表面积 = `πrl`
    - 表面积 = `πr(r + l)`
    - 体积 = `πr²h / 3`

- `三角形面积`
  - 输入：高度、下底
  - 输出：三角形面积
  - 公式：
    - 面积 = `底 × 高 / 2`

- `矩形`
  - 输入：长度、宽度
  - 输出：面积、周长
  - 公式：
    - 面积 = `长 × 宽`
    - 周长 = `2 × (长 + 宽)`

- `圆形`
  - 输入：半径
  - 输出：面积、周长
  - 公式：
    - 面积 = `πr²`
    - 周长 = `2πr`

- `梯形`
  - 输入：上底、下底、高
  - 输出：面积
  - 公式：
    - 面积 = `(上底 + 下底) × 高 / 2`

**Interaction Rules**

- 输入框仅接收数字型文本，允许小数点。
- 为空、无法解析或数值小于等于 0 时，不执行计算，给出友好提示。
- 结果默认在点击“开始计算”后更新。
- 结果展示沿用 Figma 的字段布局和单位：
  - 长度使用 `m`
  - 面积使用 `㎡`
  - 体积使用 `m³`

**Entry Flow**

- 首页点击“计算器”
- 进入计算器总页
- 点击任一 item
- 跳转到对应详情页
- 输入参数后点击“开始计算”
- 在结果卡片中展示计算结果

**Success Criteria**

- 首页“计算器”入口不再弹提示，而是进入新的计算器页。
- 计算器总页视觉、标题、卡片布局基本对齐 Figma。
- 8 个 item 均可进入详情页。
- 8 个详情页均支持有效输入和正确结果输出。
- 核心公式有单元测试覆盖。

**Risks**

- 现有工程已存在未提交改动，本次实现需要避免误覆盖用户工作。
- `TextField` 的数字输入在不同地区小数点格式上可能有差异，第一版按当前中文场景使用英文小数点。
- Figma 为静态稿，键盘弹起后的细节滚动行为需在实现时做合理补齐。

**Out of Scope**

- 计算历史记录
- 单位切换
- 导出、分享结果
- 复杂科学计算
